  • React 框架造轮子

    • 该项目使用 React 、Webpack和 TypeScript 搭建的一个造轮子项目。目前有 icon、dialog、Layout 三个组件。每个组件均经历从需求分析、调试、单元测试。完成该项目使我对 React 的常用特征更加熟悉，同时提升了自己对前端工程化流程的深刻理解，近一步了解开源文化。
    • 使用的技术栈：React、TypeScript、Webpack。

      • 技术难点：
      • TypeScript 的兼容性问题：
            重复的标识符LibraryManagedAttributes, 这是因为 yarn 解析 @types/react 的版本冲突问题。
        "skipLibCheck": true: 忽略所有声明文件的类型检查

      • 在页面中展示高亮代码：
            可以使用 require 引用 url, 直接展示指定的页面内容。
        如下面一个例子
        const x= require(`!!raw-loader!${props.path}`
        此时又有一个问题, require 不能动态地展示地址, 即现在只能在每个需要的地方手动添加需要的地址。

      • 组件的样式问题:
            在 dialog 组件中使用内联样式会出现一部分没有遮拦的情况, 需要也将这部分包裹住, 此时出现两个同级的 div , 用 Fragment 标签将内容包裹起来。
            Fragment 作用: 允许将子列表分组，无需向 DOM 添加额外节点。
            然后使用传送门 Portal, 动态生成组件，最后闭包传 API。
            Portal 用法: 当从组件的 render 方法返回一个元素时，该元素将被挂载到 DOM 节点中离其最近的父节点。
